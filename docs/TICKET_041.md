# TICKET-041 Implement: Frontend Growth History

다음 가이드라인에 따라 식물 성장 이력 타임라인 UI를 구현해줘. 전체 문서를 읽지 말고 이 요약에만 집중해.

### 1. 목표 및 핵심 로직
- **목표**: 식물의 과거 이벤트(물주기, 노트, 환경 알림, 기분 변화)를 역순(최신순) 타임라인으로 표시.
- **핵심 API 연동**:
  - `GET /plants/{plant_id}/history`: 해당 식물의 모든 이력 데이터 수신.
- **데이터 정렬**: 백엔드 응답을 `timestamp` 기준으로 내림차순 정렬하여 렌더링.

### 2. 주요 UI 컴포넌트 및 기능
1. **GrowthHistoryScreen**: 
   - **FilterTabs**: '전체', '관리', '환경', '상태' 카테고리별 필터링 기능.
2. **GrowthTimeline**: 
   - 세로 선(Line)을 따라 이벤트가 나열되는 타임라인 컨테이너.
3. **HistoryTimelineItem**: 
   - **유형별 배지(Badge)**: 'Care', 'Env', 'Status'를 아이콘이나 색상으로 구분.
   - **내용 표시**: "물주기 완료", "습도가 너무 낮음", "행복함" 등 이벤트 메시지와 발생 시각 표시.
4. **EmptyState**: 이력이 없을 때 "아직 기록된 이력이 없어요" 메시지 표시.

### 3. 필수 구현 대상
- **API Integration**: `X-User-Id` 헤더를 포함하여 이력 조회 API 호출.
- **Type Mapping**: 백엔드 응답의 `event_type`에 따라 적절한 UI 테마(색상, 아이콘) 매칭.
- **Loading/Error State**: 데이터 페칭 중 로딩 스피너 및 에러 메시지 처리.

### 4. 제약 사항 (절대 금지)
- **그래프 구현 금지**: 이번 티켓에서는 선 그래프(Line Chart)나 막대 그래프를 그리지 마. (리스트/타임라인 중심)
- **사진 타임라인 금지**: 식물 사진 변천사나 타임랩스 기능을 구현하지 마.
- **데이터 수정 금지**: 이력 화면에서 기록을 삭제하거나 수정하는 기능을 만들지 마. (Read-only)
- **백엔드 수정 금지**: 기존 API 응답 구조나 DB 로직을 변경하지 마.

지금 바로 `src/pages/GrowthHistory/` 경로에 타임라인 컴포넌트 작성을 시작해줘.
