# TICKET-040 Implement: Frontend Chat + Answer View

다음 가이드라인에 따라 지능형 채팅 및 답변 상세 UI를 구현해줘. 전체 문서를 읽지 말고 이 요약에만 집중해.

### 1. 목표 및 핵심 로직
- **목표**: 식물 케어, 병충해 상담, 동반 식물 추천 질문을 수행하고 고정 형식의 답변을 렌더링.
- **핵심 API 연동**:
  - `POST /plants/{plant_id}/chat`: 질문(prompt)을 전송하고 정형화된 답변 객체 수신.
- **고정 답변 렌더링**: 수신된 `parsed_sections` 데이터를 기반으로 [결론], [근거], [행동], [주의]를 시각적으로 구분하여 표시.

### 2. 주요 UI 컴포넌트 및 기능
1. **ChatScreen**: 
   - 질문 프리셋 바(`QuestionPresetBar`): "물 언제 줘야 해?", "잎이 노란색이야" 등 자주 묻는 질문 버튼.
   - 채팅 입력창 및 전송 로직.
2. **FixedAnswerView**: 
   - 4개 섹션을 카드 또는 아코디언 형태로 구조화하여 렌더링.
3. **Intent-specific UI**: 
   - **병충해(Pest)**: 답변에 병충해 관련 내용이 있으면 상단에 빨간색 `PestCautionBanner` 노출.
   - **동반 식물(Companion)**: 추천 식물 리스트가 포함된 경우 인라인 카드 형태로 표시.
4. **EvidenceSummary**: 
   - 답변 하단에 "참고한 정보: [센서 데이터], [식물 도감]" 등의 근거 요약 표시.

### 3. 필수 구현 대상
- **API Integration**: `X-User-Id` 헤더와 `plant_id`를 포함하여 채팅 요청 수행.
- **State Management**: 로딩 중(Thinking...), 에러 발생, 답변 완료 상태 관리.
- **Scroll Logic**: 새로운 답변이 오면 자동으로 하단 스크롤 처리.

### 4. 제약 사항 (절대 금지)
- **실시간 스트리밍 금지**: SSE나 WebSocket을 통한 글자 단위 스트리밍 구현 금지 (한 번에 응답 처리).
- **브라우저 LLM 금지**: 프론트엔드에서 직접 LLM SDK를 호출하거나 API 키를 노출하지 마.
- **이미지 진단 UI 금지**: 이미지 업로드 버튼이나 이미지 분석 결과 화면은 여기서 만들지 마.
- **백엔드 수정 금지**: 기존 API 응답 구조를 변경하지 마.

지금 바로 `src/pages/Chat/` 경로에 채팅 화면 및 답변 렌더러 작성을 시작해줘.
